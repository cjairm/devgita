# i3 Module Documentation

## Overview

The i3 module provides installation and configuration management for i3 window manager with devgita integration. It follows the standardized devgita app interface while providing i3-specific operations for tiling window management and workspace configuration.

## App Purpose

i3 is a tiling window manager designed for X11, targeted at developers and advanced users. This module ensures i3 is properly installed and configured with devgita's optimized settings for development workflows, including vim-style keybindings, workspace management, and Gruvbox theming. It serves as the Debian/Ubuntu equivalent to Aerospace on macOS.

## Lifecycle Summary

1. **Installation**: Install i3 package via apt package manager
2. **Configuration**: Apply devgita's i3 configuration templates with developer-focused settings
3. **Execution**: i3 commands available for window management automation (currently placeholder)

## Exported Functions

| Function           | Purpose                   | Behavior                                                             |
| ------------------ | ------------------------- | -------------------------------------------------------------------- |
| `New()`            | Factory method            | Creates new i3 instance with platform-specific commands              |
| `Install()`        | Standard installation     | Uses `InstallPackage()` to install i3                                |
| `ForceInstall()`   | Force installation        | Calls `Uninstall()` first (returns error if fails), then `Install()` |
| `SoftInstall()`    | Conditional installation  | Uses `MaybeInstallPackage()` to check before installing              |
| `ForceConfigure()` | Force configuration       | Overwrites existing i3 config with devgita defaults                  |
| `SoftConfigure()`  | Conditional configuration | Preserves existing config file if present                            |
| `Uninstall()`      | Remove installation       | **Not supported** - returns error                                    |
| `ExecuteCommand()` | Execute i3 commands       | Placeholder for window management automation                         |
| `Update()`         | Update installation       | **Not implemented** - returns error                                  |

## Installation Methods

### Install()

```go
i := i3.New()
err := i.Install()
```

- **Purpose**: Standard i3 installation
- **Behavior**: Uses `InstallPackage()` to install i3 package
- **Use case**: Initial i3 installation or explicit reinstall

### SoftInstall()

```go
i := i3.New()
err := i.SoftInstall()
```

- **Purpose**: Install i3 only if not already present
- **Behavior**: Uses `MaybeInstallPackage()` to check before installing
- **Use case**: Standard installation that respects existing i3 installations

### ForceInstall()

```go
i := i3.New()
err := i.ForceInstall()
```

- **Purpose**: Force i3 installation regardless of existing state
- **Behavior**: Calls `Uninstall()` first (returns error if it fails), then `Install()`
- **Use case**: Ensure fresh i3 installation or fix corrupted installation

### Uninstall()

```go
err := i.Uninstall()
```

- **Purpose**: Remove i3 installation
- **Behavior**: **Not supported** - returns error
- **Rationale**: Window managers are typically managed at the system level

### Update()

```go
err := i.Update()
```

- **Purpose**: Update i3 installation
- **Behavior**: **Not implemented** - returns error
- **Rationale**: i3 updates are typically handled by the system package manager

## Configuration Methods

### Configuration Paths

- **Source**: `paths.Paths.App.Configs.I3` (devgita's i3 configs)
- **Destination**: `paths.Paths.Config.I3` (user's i3 config directory)
- **Marker file**: `config` in i3 config directory

### ForceConfigure()

```go
err := i.ForceConfigure()
```

- **Purpose**: Apply i3 configuration regardless of existing files
- **Behavior**: Copies all configs from app dir to local dir, overwriting existing
- **Use case**: Reset to devgita defaults, apply config updates

### SoftConfigure()

```go
err := i.SoftConfigure()
```

- **Purpose**: Apply i3 configuration only if not already configured
- **Behavior**: Checks for `config` marker file; if exists, does nothing
- **Marker logic**: `filepath.Join(paths.Paths.Config.I3, "config")`
- **Use case**: Initial setup that preserves user customizations

## Execution Methods

### ExecuteCommand()

```go
err := i.ExecuteCommand("reload")
err := i.ExecuteCommand("restart")
```

- **Purpose**: Execute i3 commands for automation
- **Parameters**: Variable arguments (currently placeholder)
- **Behavior**: Returns nil (placeholder for future window management automation)
- **Use case**: Future window management automation scripts

## Expected Function Interactions

1. **Standard Setup**: `New()` → `SoftInstall()` → `SoftConfigure()`
2. **Force Setup**: `New()` → `ForceInstall()` → `ForceConfigure()`
3. **Update Configuration**: `New()` → `SoftInstall()` → `ForceConfigure()`

## Constants and Paths

### Relevant Constants

- `constants.I3`: Package name ("i3") for installation
- Used by all installation methods for consistent package reference

### Configuration Paths

- `paths.Paths.App.Configs.I3`: Source directory for devgita's i3 configuration templates
- `paths.Paths.Config.I3`: Target directory for user's i3 configuration
- Configuration copying preserves directory structure and file permissions

## Implementation Notes

- **ForceInstall Logic**: Calls `Uninstall()` first and returns the error if it fails since i3 uninstall is not supported
- **Configuration Strategy**: Uses marker file (`config`) to determine if configuration exists
- **Error Handling**: All methods return errors that should be checked by callers
- **Platform Independence**: Uses command interface abstraction for cross-platform compatibility
- **Update Method**: Not implemented as i3 updates should be handled by system package managers

## Configuration Structure

The i3 configuration (`config`) includes developer-focused enhancements:

### Basic Settings

```bash
# Set mod key to Super (Windows key)
set $mod Mod4

# Font for window titles and bar
font pango:JetBrains Mono 10

# Use Mouse+$mod to drag floating windows
floating_modifier $mod
```

### Keybindings

```bash
# Terminal
bindsym $mod+Return exec alacritty

# Kill focused window
bindsym $mod+Shift+q kill

# Reload configuration
bindsym $mod+Shift+c reload

# Restart i3 inplace
bindsym $mod+Shift+r restart

# Vim-style focus navigation
bindsym $mod+h focus left
bindsym $mod+j focus down
bindsym $mod+k focus up
bindsym $mod+l focus right

# Vim-style window movement
bindsym $mod+Shift+h move left
bindsym $mod+Shift+j move down
bindsym $mod+Shift+k move up
bindsym $mod+Shift+l move right
```

### Workspace Management

```bash
# Define workspaces
set $ws1 "1"
set $ws2 "2"
set $ws3 "3"
# ... up to $ws9 "9"

# Switch to workspace
bindsym $mod+1 workspace $ws1
bindsym $mod+2 workspace $ws2
# ... up to $mod+9

# Move container to workspace
bindsym $mod+Shift+1 move container to workspace $ws1
bindsym $mod+Shift+2 move container to workspace $ws2
# ... up to $mod+Shift+9
```

### Layout Settings

```bash
# Split orientation
bindsym $mod+v split h
bindsym $mod+s split v

# Fullscreen
bindsym $mod+f fullscreen toggle

# Change layout
bindsym $mod+e layout toggle split
bindsym $mod+w layout tabbed

# Toggle floating
bindsym $mod+Shift+space floating toggle

# Focus floating/tiling
bindsym $mod+space focus mode_toggle
```

### Appearance

```bash
# Window gaps (requires i3-gaps)
gaps inner 10
gaps outer 5

# Window borders
default_border pixel 2
default_floating_border pixel 2

# Gruvbox theme colors
set $bg #282828
set $fg #ebdbb2
set $red #cc241d
set $green #98971a
set $yellow #d79921
set $blue #458588
set $purple #b16286
set $aqua #689d68
set $gray #a89984
set $darkgray #1d2021

# Window colors
#                       border    background text    indicator
client.focused          $blue     $blue      $bg     $purple
client.focused_inactive $darkgray $darkgray  $fg     $purple
client.unfocused        $darkgray $darkgray  $gray   $purple
client.urgent           $red      $red       $fg     $red
```

### Status Bar

```bash
bar {
    status_command i3status
    position top
    
    colors {
        background $bg
        statusline $fg
        separator  $gray
        
        #                  border    background text
        focused_workspace  $blue     $blue      $bg
        active_workspace   $darkgray $darkgray  $fg
        inactive_workspace $darkgray $darkgray  $gray
        urgent_workspace   $red      $red       $fg
    }
}
```

## Troubleshooting

### Common Issues

1. **Installation Fails**: Ensure apt is available and updated
2. **Configuration Not Applied**: Check file permissions in config directory
3. **i3 Won't Start**: Verify X11 display server is running
4. **Keybindings Not Working**: Check for conflicts with system shortcuts

### Platform Considerations

- **Debian/Ubuntu**: Installed via apt package manager
- **Configuration Location**: `~/.config/i3/config`
- **Display Server**: Requires X11 (not compatible with Wayland natively)
- **Dependencies**: May require i3status, dmenu, or other tools

### Prerequisites

Before using i3:

- **Linux**: Debian 12+ (Bookworm) or Ubuntu 24+ with X11
- **X11**: Functional X11 display server
- **Login Manager**: Display manager that supports i3 session selection

## Integration with Devgita

i3 integrates with devgita's desktop category:

- **Installation**: Installed as part of desktop applications setup (Debian/Ubuntu only)
- **Configuration**: Template-based with devgita defaults
- **Usage**: Window manager for X11 sessions
- **Updates**: Managed through apt package manager
- **Category**: Window manager in desktop category
- **Platform Equivalent**: Debian/Ubuntu alternative to macOS Aerospace

## Key Features

### Tiling Window Management

- Automatic window tiling
- Manual split control (horizontal/vertical)
- Container-based layout
- Floating window support

### Workspace Management

- Multiple virtual workspaces (1-9+)
- Workspace switching with keyboard
- Move windows between workspaces
- Monitor-aware workspace assignment

### Keyboard-Driven

- Vim-style navigation (h/j/k/l)
- Configurable keybindings
- Modal operation support
- Quick window operations

### Customization

- Configuration file-based
- Gruvbox theme integration
- Gap support (i3-gaps)
- Status bar customization

### Development Use Cases

- Efficient screen space usage
- Quick window navigation
- Multiple workspace organization
- Terminal-focused workflow
- Keyboard-driven productivity
